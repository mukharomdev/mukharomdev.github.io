f:T1590,<p>Git adalah sistem kontrol versi yang digunakan untuk melacak perubahan dalam kode sumber selama pengembangan perangkat lunak. Berikut adalah panduan langkah demi langkah untuk menggunakan Git:</p>
<hr>
<h3><strong>1. Instal Git</strong></h3>
<ul>
<li><strong>Windows</strong>: Unduh Git dari <a href="https://git-scm.com/">git-scm.com</a> dan ikuti petunjuk instalasi.</li>
<li><strong>Mac</strong>: Gunakan Homebrew (brew install git) atau unduh dari situs resmi.</li>
<li><strong>Linux</strong>: Gunakan manajer paket (misalnya, sudo apt install git untuk Ubuntu).</li>
</ul>
<p>Verifikasi instalasi:</p>
<pre><code class="language-bash">git --version
</code></pre>
<hr>
<h3><strong>2. Konfigurasi Git</strong></h3>
<p>Atur nama pengguna dan email (digunakan untuk commit):</p>
<pre><code class="language-bash">git config --global user.name "Nama Anda"
git config --global user.email "email.anda@example.com"
</code></pre>
<p>Periksa konfigurasi Anda:</p>
<pre><code class="language-bash">git config --list
</code></pre>
<hr>
<h3><strong>3. Buat Repositori Baru</strong></h3>
<p>Inisialisasi repositori Git baru di folder proyek Anda:</p>
<pre><code class="language-bash">git init
</code></pre>
<p>Ini akan membuat folder .git yang tersembunyi untuk melacak perubahan.</p>
<hr>
<h3><strong>4. Periksa Status Repositori</strong></h3>
<p>Lihat status saat ini dari repositori Anda:</p>
<pre><code class="language-bash">git status
</code></pre>
<hr>
<h3><strong>5. Tambahkan File ke Staging Area</strong></h3>
<p>Stage file untuk mempersiapkannya untuk commit:</p>
<pre><code class="language-bash">git add &#x3C;nama-file>
</code></pre>
<p>Untuk menambahkan semua file:</p>
<pre><code class="language-bash">git add .
</code></pre>
<hr>
<h3><strong>6. Commit Perubahan</strong></h3>
<p>Simpan perubahan Anda dengan pesan commit:</p>
<pre><code class="language-bash">git commit -m "Pesan commit Anda"
</code></pre>
<hr>
<h3><strong>7. Lihat Riwayat Commit</strong></h3>
<p>Lihat log semua commit:</p>
<pre><code class="language-bash">git log
</code></pre>
<hr>
<h3><strong>8. Buat Branch</strong></h3>
<p>Branch memungkinkan Anda bekerja pada versi berbeda dari proyek Anda:</p>
<pre><code class="language-bash">git branch &#x3C;nama-branch>
</code></pre>
<p>Beralih ke branch:</p>
<pre><code class="language-bash">git checkout &#x3C;nama-branch>
</code></pre>
<p>Buat dan beralih ke branch baru dalam satu perintah:</p>
<pre><code class="language-bash">git checkout -b &#x3C;nama-branch>
</code></pre>
<hr>
<h3><strong>9. Gabungkan Branch</strong></h3>
<p>Gabungkan perubahan dari satu branch ke branch lain:</p>
<pre><code class="language-bash">git checkout main
git merge &#x3C;nama-branch>
</code></pre>
<hr>
<h3><strong>10. Repositori Remote (GitHub, GitLab, dll.)</strong></h3>
<ul>
<li><strong>Clone repositori</strong>:
<pre><code class="language-bash">git clone &#x3C;url-repositori>
</code></pre>
</li>
<li><strong>Tambahkan remote</strong>:
<pre><code class="language-bash">git remote add origin &#x3C;url-repositori>
</code></pre>
</li>
<li><strong>Push perubahan</strong>:
<pre><code class="language-bash">git push origin &#x3C;nama-branch>
</code></pre>
</li>
<li><strong>Pull perubahan</strong>:
<pre><code class="language-bash">git pull origin &#x3C;nama-branch>
</code></pre>
</li>
</ul>
<hr>
<h3><strong>11. Selesaikan Konflik</strong></h3>
<p>Saat menggabungkan branch, konflik mungkin terjadi. Buka file yang bermasalah, selesaikan perbedaan, lalu:</p>
<pre><code class="language-bash">git add &#x3C;nama-file>
git commit
</code></pre>
<hr>
<h3><strong>12. Batalkan Perubahan</strong></h3>
<ul>
<li><strong>Unstage file</strong>:
<pre><code class="language-bash">git reset &#x3C;nama-file>
</code></pre>
</li>
<li><strong>Kembalikan ke commit terakhir</strong>:
<pre><code class="language-bash">git checkout -- &#x3C;nama-file>
</code></pre>
</li>
<li><strong>Ubah commit terakhir</strong>:
<pre><code class="language-bash">git commit --amend
</code></pre>
</li>
</ul>
<hr>
<h3><strong>13. Tagging</strong></h3>
<p>Buat tag untuk commit tertentu (misalnya, untuk rilis):</p>
<pre><code class="language-bash">git tag &#x3C;nama-tag>
</code></pre>
<p>Push tag ke repositori remote:</p>
<pre><code class="language-bash">git push origin --tags
</code></pre>
<hr>
<h3><strong>14. Abaikan File</strong></h3>
<p>Buat file .gitignore untuk mengecualikan file/folder dari pelacakan:</p>
<pre><code class="language-plaintext"># Contoh file .gitignore
node_modules/
*.log
</code></pre>
<hr>
<h3><strong>15. Perintah Lanjutan</strong></h3>
<ul>
<li><strong>Stash perubahan</strong>: Simpan sementara perubahan tanpa commit:
<pre><code class="language-bash">git stash
</code></pre>
</li>
<li><strong>Rebase</strong>: Terapkan ulang commit di atas branch lain:
<pre><code class="language-bash">git rebase &#x3C;nama-branch>
</code></pre>
</li>
<li><strong>Cherry-pick</strong>: Terapkan commit tertentu dari branch lain:
<pre><code class="language-bash">git cherry-pick &#x3C;hash-commit>
</code></pre>
</li>
</ul>
<hr>
<h3><strong>16. Praktik Terbaik</strong></h3>
<ul>
<li>Tulis pesan commit yang jelas dan deskriptif.</li>
<li>Commit sering dan dalam bagian kecil.</li>
<li>Gunakan branch untuk fitur baru atau perbaikan bug.</li>
<li>Secara teratur pull perubahan dari repositori remote untuk menghindari konflik.</li>
</ul>
<hr>
