f:T162b,<p>useImperativeHandle adalah hook di React yang digunakan untuk <strong>menyesuaikan nilai instance (metode atau properti) yang diekspos ke komponen parent</strong> ketika menggunakan ref pada komponen anak. Hook ini umumnya dipakai bersama forwardRef untuk mengontrol perilaku komponen anak secara imperatif (seperti memanggil fungsi atau mengakses nilai langsung).</p>
<hr>
<h3><strong>Kapan Menggunakan useImperativeHandle?</strong></h3>
<ol>
<li>Saat ingin <strong>membatasi akses parent</strong> ke metode/properti tertentu dari komponen anak.</li>
<li>Untuk <strong>integrasi dengan library eksternal</strong> (misalnya, mengontrol DOM element secara imperatif).</li>
<li>Ketika perlu <strong>mengakses fungsi komponen anak secara langsung</strong> (contoh: fokus input, play/pause video).</li>
</ol>
<hr>
<h3><strong>Sintaks Dasar</strong></h3>
<pre><code class="language-javascript">useImperativeHandle(ref, () => ({
  // Metode/properti yang ingin diekspos ke parent
}), [dependencies]);
</code></pre>
<hr>
<h3><strong>Contoh Penggunaan</strong></h3>
<h4>1. <strong>Komponen Anak dengan Metode Imperatif</strong></h4>
<pre><code class="language-jsx">import { forwardRef, useImperativeHandle, useRef } from 'react';

// Komponen anak menggunakan forwardRef
const CustomInput = forwardRef((props, ref) => {
  const inputRef = useRef();

  // Ekspos metode focus dan getValue ke parent
  useImperativeHandle(ref, () => ({
    focus: () => inputRef.current.focus(),
    getValue: () => inputRef.current.value,
  }), []); // Dependency array kosong ‚Üí hanya dijalankan sekali

  return &#x3C;input ref={inputRef} {...props} />;
});

// Komponen Parent
function Parent() {
  const inputRef = useRef();

  const handleClick = () => {
    inputRef.current.focus(); // Panggil metode focus dari parent
    console.log('Nilai input:', inputRef.current.getValue());
  };

  return (
    &#x3C;div>
      &#x3C;CustomInput ref={inputRef} />
      &#x3C;button onClick={handleClick}>Fokus Input&#x3C;/button>
    &#x3C;/div>
  );
}
</code></pre>
<h4>2. <strong>Integrasi dengan Video Player</strong></h4>
<pre><code class="language-jsx">const VideoPlayer = forwardRef((props, ref) => {
  const videoRef = useRef();

  useImperativeHandle(ref, () => ({
    play: () => videoRef.current.play(),
    pause: () => videoRef.current.pause(),
    isPlaying: () => !videoRef.current.paused,
  }), []);

  return &#x3C;video ref={videoRef} src="video.mp4" />;
});

function App() {
  const playerRef = useRef();

  return (
    &#x3C;div>
      &#x3C;VideoPlayer ref={playerRef} />
      &#x3C;button onClick={() => playerRef.current.play()}>Play&#x3C;/button>
      &#x3C;button onClick={() => playerRef.current.pause()}>Pause&#x3C;/button>
    &#x3C;/div>
  );
}
</code></pre>
<hr>
<h3><strong>Aturan Penting</strong></h3>
<ol>
<li>
<p><strong>Wajib Digunakan dengan forwardRef</strong><br>
Komponen anak harus dibungkus dengan forwardRef untuk menerima ref dari parent.</p>
</li>
<li>
<p><strong>Hanya Ekspos yang Diperlukan</strong><br>
Batasi metode/properti yang diekspos untuk mengurangi kompleksitas.</p>
</li>
<li>
<p><strong>Perhatikan Dependency Array</strong><br>
Jika metode bergantung pada nilai state/props, tambahkan ke dependency array:</p>
<pre><code class="language-javascript">const [count, setCount] = useState(0);

useImperativeHandle(ref, () => ({
  getCount: () => count, // Perbarui saat &#x3C;code>count&#x3C;/code> berubah
}), [count]);
</code></pre>
</li>
</ol>
<hr>
<h3><strong>Perbandingan useImperativeHandle vs. ref Biasa</strong></h3>
<div class="overflow-x-auto bg-lime-100"><table>
<thead>
<tr>
<th><strong>Kriteria</strong></th>
<th>useImperativeHandle</th>
<th>ref Biasa</th>
</tr>
</thead>
<tbody>
<tr>
<td><strong>Akses ke Instance</strong></td>
<td>Terbatas (hanya yang diekspos)</td>
<td>Full akses ke DOM/node</td>
</tr>
<tr>
<td><strong>Kontrol</strong></td>
<td>Parent mengontrol secara terbatas</td>
<td>Parent bisa manipulasi langsung</td>
</tr>
<tr>
<td><strong>Kasus Penggunaan</strong></td>
<td>Komponen dengan API spesifik</td>
<td>Integrasi DOM/library eksternal</td>
</tr>
</tbody>
</table></div>
<hr>
<h3><strong>Best Practices</strong></h3>
<ul>
<li><strong>Gunakan untuk Kasus Spesifik</strong>: Hindari penggunaan berlebihan. Sebisa mungkin, gunakan komunikasi data via props/state.</li>
<li><strong>Encapsulation</strong>: Sembunyikan detail internal komponen anak.</li>
<li><strong>Type Safety (TypeScript)</strong>: Jika menggunakan TypeScript, definisikan tipe untuk ref:
<pre><code class="language-typescript">type VideoPlayerHandle = {
  play: () => void;
  pause: () => void;
  isPlaying: () => boolean;
};

const VideoPlayer = forwardRef&#x3C;VideoPlayerHandle>((props, ref) => {
  // ...
});
</code></pre>
</li>
</ul>
<hr>
<h3><strong>Kesalahan Umum</strong></h3>
<ol>
<li>
<p><strong>Lupa Dependency Array</strong><br>
Jika metode bergantung pada nilai yang berubah, pastikan untuk menambahkannya ke dependency array untuk menghindari <em>stale closure</em>.</p>
</li>
<li>
<p><strong>Ekspos Seluruh Instance</strong><br>
Jangan mengekspos semua metode DOM (misal: ref.current langsung), karena bisa menyebabkan ketidakkonsistenan.</p>
</li>
</ol>
<hr>
<h3><strong>Kesimpulan</strong></h3>
<p>useImperativeHandle berguna ketika:</p>
<ul>
<li>Parent perlu <strong>mengontrol komponen anak secara imperatif</strong>.</li>
<li>Anda ingin <strong>membatasi akses parent</strong> ke fungsi tertentu.</li>
<li>Berintegrasi dengan <strong>library pihak ketiga yang memerlukan ref</strong>.</li>
</ul>
