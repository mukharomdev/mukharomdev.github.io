f:T16e9,<p>useSyncExternalStore adalah hook di React yang memungkinkan komponen untuk <strong>berlangganan (subscribe) ke sumber data eksternal</strong> di luar React (seperti store Redux, Zustand, atau browser APIs) dan tetap sinkron dengan perubahan yang terjadi. Hook ini sangat berguna untuk integrasi dengan sistem manajemen state eksternal atau data yang berubah di luar siklus hidup React.</p>
<hr>
<h3><strong>Kapan Menggunakan useSyncExternalStore?</strong></h3>
<ol>
<li><strong>Integrasi dengan state eksternal</strong> (contoh: global store seperti Redux).</li>
<li><strong>Berlangganan ke browser APIs</strong> (seperti geolocation, online status, atau media queries).</li>
<li><strong>Menggunakan data dari WebSocket atau event-driven sources</strong>.</li>
<li><strong>Menghindari masalah "tearing"</strong> (ketidaksesuaian tampilan) saat menggunakan React Concurrent Features.</li>
</ol>
<hr>
<h3><strong>Sintaks Dasar</strong></h3>
<pre><code class="language-javascript">const state = useSyncExternalStore(
  subscribe, // Fungsi untuk subscribe ke perubahan
  getSnapshot, // Fungsi untuk mengambil nilai saat ini
  getServerSnapshot? // (Opsional) Fungsi untuk SSR
);
</code></pre>
<ul>
<li><strong>subscribe</strong>: Fungsi yang mendaftarkan callback untuk dipanggil saat data eksternal berubah.</li>
<li><strong>getSnapshot</strong>: Fungsi yang mengembalikan nilai terbaru dari data eksternal.</li>
<li><strong>getServerSnapshot</strong>: Fungsi untuk mengambil nilai awal saat rendering di server (SSR).</li>
</ul>
<hr>
<h3><strong>Contoh Penggunaan</strong></h3>
<h4>1. <strong>Subscribe ke Store Eksternal Sederhana</strong></h4>
<pre><code class="language-javascript">// External store (contoh sederhana)
let counter = 0;
const listeners = new Set();

function increment() {
  counter++;
  listeners.forEach((listener) => listener());
}

function subscribe(callback) {
  listeners.add(callback);
  return () => listeners.delete(callback); // Cleanup subscription
}

function getSnapshot() {
  return counter;
}

// Komponen React
function Counter() {
  const count = useSyncExternalStore(subscribe, getSnapshot);

  return (
    &#x3C;div>
      &#x3C;p>Count: {count}&#x3C;/p>
      &#x3C;button onClick={increment}>+&#x3C;/button>
    &#x3C;/div>
  );
}
</code></pre>
<h4>2. <strong>Subscribe ke Browser API (Online Status)</strong></h4>
<pre><code class="language-javascript">function OnlineStatus() {
  const isOnline = useSyncExternalStore(
    (callback) => {
      window.addEventListener('online', callback);
      window.addEventListener('offline', callback);
      return () => {
        window.removeEventListener('online', callback);
        window.removeEventListener('offline', callback);
      };
    },
    () => navigator.onLine // getSnapshot
  );

  return &#x3C;div>Status: {isOnline ? 'Online' : 'Offline'}&#x3C;/div>;
}
</code></pre>
<h4>3. <strong>Dengan Server-Side Rendering (SSR)</strong></h4>
<pre><code class="language-javascript">function StoreData() {
  const data = useSyncExternalStore(
    subscribe,
    getSnapshot,
    // getServerSnapshot (untuk SSR)
    () => 'Nilai awal dari server'
  );

  return &#x3C;div>{data}&#x3C;/div>;
}
</code></pre>

<div class="overflow-x-auto bg-lime-100"><table>

<td>Mendukung dengan getServerSnapshot</td>
<td>Membutuhkan penanganan manual</td>
</tr>

<hr>
<h3><strong>Best Practices</strong></h3>
<ol>
<li><strong>Gunakan untuk Sumber Data Eksternal</strong>: Jangan digunakan untuk state lokal komponen.</li>
<li><strong>Optimasi dengan useMemo</strong>: Jika subscribe atau getSnapshot kompleks, gunakan useMemo untuk mencegah re-render tidak perlu.
<pre><code class="language-javascript">const subscribe = useMemo(() => (callback) => {
  // Logika subscribe
}, []);
</code></pre>
</li>
<li><strong>Hindari Side Effects di getSnapshot</strong>: Fungsi ini harus murni (pure function).</li>
<li><strong>Handle Error dengan Baik</strong>: Pastikan sumber data eksternal memiliki penanganan error.</li>
</ol>
<hr>
<h3><strong>Pitfall (Kesalahan Umum)</strong></h3>
<ul>
<li><strong>Memory Leak</strong>: Lupa melakukan unsubscribe di cleanup function.</li>
<li><strong>Stale Closure</strong>: Pastikan getSnapshot selalu mengembalikan nilai terbaru.</li>
<li><strong>Tidak Support SSR</strong>: Jika lupa menyediakan getServerSnapshot, mungkin menyebabkan hydration mismatch.</li>
</ul>
<hr>
<h3><strong>Contoh Integrasi dengan Zustand Store</strong></h3>
<pre><code class="language-javascript">import { useSyncExternalStore } from 'react';
import create from 'zustand';

const useStore = create((set) => ({
  user: null,
  login: (user) => set({ user }),
  logout: () => set({ user: null }),
}));

function UserProfile() {
  const user = useSyncExternalStore(
    useStore.subscribe,
    () => useStore.getState().user
  );

  return &#x3C;div>{user ? user.name : 'Belum login'}&#x3C;/div>;
}
</code></pre>
<hr>
<h3><strong>Kesimpulan</strong></h3>
<p>useSyncExternalStore adalah hook yang powerful untuk:</p>
<ul>
<li><strong>Integrasi dengan sistem state eksternal</strong> (Redux, Zustand, dll).</li>
<li><strong>Subscribe ke data dinamis dari browser APIs</strong>.</li>
<li><strong>Mendukung SSR dengan aman</strong> melalui getServerSnapshot.</li>
</ul>
<p>Gunakan hook ini ketika Anda perlu <strong>menghubungkan React dengan sumber data di luar ekosistem React</strong>! 🚀</p>
