f:T1861,<p>useDeferredValue adalah hook di React yang digunakan untuk <strong>menunda pembaruan nilai</strong> yang tidak prioritas/urgent, sehingga memungkinkan komponen untuk tetap responsif saat menangani operasi berat atau pembaruan yang lambat. Hook ini sering digunakan bersama dengan Suspense dan fitur Concurrent Mode untuk mengoptimalkan performa UI.</p>
<hr>
<h3><strong>1. Kapan Menggunakan useDeferredValue?</strong></h3>
<ul>
<li><strong>Menunda render konten yang berat</strong> (misal: hasil pencarian, daftar besar).</li>
<li><strong>Mengoptimalkan responsivitas input</strong> (misal: input pencarian yang memicu render data besar).</li>
<li><strong>Mencegah UI "lag"</strong> saat menunggu operasi asinkron (seperti fetching data).</li>
</ul>
<hr>
<h3><strong>2. Sintaks Dasar</strong></h3>
<pre><code class="language-javascript">const deferredValue = useDeferredValue(value);
</code></pre>
<ul>
<li><strong>value</strong>: Nilai yang ingin ditunda pembaruannya.</li>
<li><strong>deferredValue</strong>: Nilai yang di-defer (bisa sama dengan value atau nilai sebelumnya selama pembaruan tertunda).</li>
</ul>
<hr>
<h3><strong>3. Contoh Penggunaan</strong></h3>
<h4>a. <strong>Input Pencarian dengan Hasil yang Ditunda</strong></h4>
<pre><code class="language-jsx">import { useState, useDeferredValue, memo } from 'react';

function SearchResults({ query }) {
  // Simulasi data besar (misal: 10.000 item)
  const items = Array(10000).fill(null).map((_, i) => &#x3C;code>Item ${i + 1} untuk "${query}"&#x3C;/code>);

  return (
    &#x3C;ul>
      {items.map((item, index) => (
        &#x3C;li key={index}>{item}&#x3C;/li>
      ))}
    &#x3C;/ul>
  );
}

// Komponen hasil pencarian di-memo untuk optimisasi
const MemoizedResults = memo(SearchResults);

function SearchApp() {
  const [query, setQuery] = useState('');
  const deferredQuery = useDeferredValue(query); // Tunda pembaruan query

  return (
    &#x3C;div>
      &#x3C;input
        type="text"
        value={query}
        onChange={(e) => setQuery(e.target.value)}
        placeholder="Cari..."
      />
      {/* Tampilkan hasil pencarian yang di-defer */}
      &#x3C;MemoizedResults query={deferredQuery} />
    &#x3C;/div>
  );
}
</code></pre>
<h4>b. <strong>Optimasi Transisi dengan useDeferredValue</strong></h4>
<pre><code class="language-jsx">function App() {
  const [text, setText] = useState('');
  const deferredText = useDeferredValue(text);

  return (
    &#x3C;div>
      &#x3C;input
        value={text}
        onChange={(e) => setText(e.target.value)}
        placeholder="Ketik sesuatu..."
      />
      {/* Komponen berat yang di-defer */}
      &#x3C;HeavyComponent text={deferredText} />
    &#x3C;/div>
  );
}

// Komponen yang render-nya lambat
const HeavyComponent = memo(({ text }) => {
  // Simulasi komputasi berat
  const items = Array(1000).fill(null).map((_, i) => text.repeat(100));

  return (
    &#x3C;div>
      {items.map((item, index) => (
        &#x3C;div key={index}>{item.substring(0, 10)}...&#x3C;/div>
      ))}
    &#x3C;/div>
  );
});
</code></pre>
<hr>
<h3><strong>4. Cara Kerja useDeferredValue</strong></h3>
<ul>
<li>Saat nilai asli (value) berubah, React akan:
<ol>
<li><strong>Render ulang komponen dengan nilai sebelumnya</strong> (stale value) untuk menjaga UI responsif.</li>
<li><strong>Render ulang di latar belakang</strong> dengan nilai baru.</li>
<li><strong>Tampilkan nilai baru</strong> begitu siap (tanpa memblokir interaksi pengguna).</li>
</ol>
</li>
</ul>
<hr>
<h3><strong>5. Perbedaan useDeferredValue vs useTransition</strong></h3>
<div class="overflow-x-auto bg-lime-100"><table>
<thead>
<tr>
<th><strong>Kriteria</strong></th>
<th>useDeferredValue</th>
<th>useTransition</th>
</tr>
</thead>
<tbody>
<tr>
<td><strong>Penggunaan</strong></td>
<td>Untuk nilai (value)</td>
<td>Untuk transisi state</td>
</tr>
<tr>
<td><strong>Contoh Kasus</strong></td>
<td>Menunda render data pencarian</td>
<td>Menunda navigasi halaman</td>
</tr>
<tr>
<td><strong>Cara Akses</strong></td>
<td>Langsung ke nilai</td>
<td>Menggunakan startTransition</td>
</tr>
<tr>
<td><strong>Prioritas</strong></td>
<td>Rendah (deferred)</td>
<td>Dapat diatur dengan prioritas</td>
</tr>
</tbody>
</table></div>

<h3><strong>6. Best Practices</strong></h3>
<ol>
<li>
<p><strong>Kombinasikan dengan memo atau useMemo</strong><br>
Untuk mencegah render ulang komponen anak yang tidak perlu:</p>
<pre><code class="language-jsx">const MemoizedComponent = memo(HeavyComponent);
</code></pre>
</li>
<li>
<p><strong>Gunakan untuk Nilai yang Sering Berubah</strong><br>
Contoh: input pencarian real-time yang memicu operasi berat.</p>
</li>
<li>
<p><strong>Hindari untuk Pembaruan Urgent</strong><br>
Misal: animasi, input yang harus langsung merespons (seperti toggle).</p>
</li>
<li>
<p><strong>Integrasi dengan Suspense</strong><br>
Untuk menampilkan fallback UI saat operasi asinkron sedang berjalan:</p>
<pre><code class="language-jsx">&#x3C;Suspense fallback={&#x3C;Spinner />}>
  &#x3C;MemoizedResults query={deferredQuery} />
&#x3C;/Suspense>
</code></pre>
</li>
</ol>
<hr>
<h3><strong>7. Contoh dengan Fetching Data</strong></h3>
<pre><code class="language-jsx">function SearchApp() {
  const [query, setQuery] = useState('');
  const deferredQuery = useDeferredValue(query);
  const results = useFetchData(deferredQuery); // Simulasi fetching data

  return (
    &#x3C;div>
      &#x3C;input
        value={query}
        onChange={(e) => setQuery(e.target.value)}
        placeholder="Cari produk..."
      />
      &#x3C;Suspense fallback={&#x3C;div>Memuat...&#x3C;/div>}>
        &#x3C;SearchResults results={results} />
      &#x3C;/Suspense>
    &#x3C;/div>
  );
}
</code></pre>

<h3><strong>Kesimpulan</strong></h3>
<p>useDeferredValue berguna untuk:</p>
<ul>
<li><strong>Meningkatkan responsivitas UI</strong> dengan menunda pembaruan yang tidak kritis.</li>
<li><strong>Mengoptimalkan render komponen berat</strong> (seperti daftar besar atau data kompleks).</li>
<li><strong>Mencegah lag</strong> saat pengguna berinteraksi dengan input yang cepat (seperti pencarian).</li>
</ul>
<p>Gunakan hook ini untuk kasus di mana <strong>pengalaman pengguna lebih penting daripada pembaruan instan</strong>! 🚀</p>
