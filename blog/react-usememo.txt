f:T15f7,<p>useMemo adalah hook di React yang digunakan untuk <strong>memoize (menyimpan cache) nilai hasil komputasi</strong>, sehingga komputasi tersebut tidak dijalankan ulang setiap kali komponen dirender, kecuali jika <em>dependency</em>-nya berubah. Ini berguna untuk mengoptimalkan performa, terutama ketika:</p>
<ol>
<li><strong>Komputasi berat</strong> (seperti filter array, kalkulasi matematika kompleks).</li>
<li><strong>Mencegah re-render komponen anak</strong> yang bergantung pada nilai tersebut.</li>
<li><strong>Mempertahankan referensi objek/array</strong> untuk mencegah efek samping di useEffect.</li>
</ol>
<hr>
<h3><strong>1. Sintaks Dasar</strong></h3>
<pre><code class="language-javascript">const memoizedValue = useMemo(
  () => computeExpensiveValue(a, b), // Fungsi yang menghasilkan nilai
  [a, b] // Dependency array (opsional)
);
</code></pre>
<hr>
<h3><strong>2. Contoh Penggunaan</strong></h3>
<h4>a. <strong>Filter Data (Optimasi Komputasi)</strong></h4>
<pre><code class="language-jsx">import { useMemo, useState } from 'react';

function ProductList({ products, searchTerm }) {
  // Hanya jalankan filter jika &#x3C;code>products&#x3C;/code> atau &#x3C;code>searchTerm&#x3C;/code> berubah
  const filteredProducts = useMemo(() => {
    return products.filter(product => 
      product.name.toLowerCase().includes(searchTerm.toLowerCase())
    );
  }, [products, searchTerm]);

  return (
    &#x3C;ul>
      {filteredProducts.map(product => (
        &#x3C;li key={product.id}>{product.name}&#x3C;/li>
      ))}
    &#x3C;/ul>
  );
}
</code></pre>
<h4>b. <strong>Mempertahankan Referensi Objek</strong></h4>
<pre><code class="language-jsx">function UserProfile({ user }) {
  // Objek config tidak dibuat ulang kecuali &#x3C;code>user.role&#x3C;/code> berubah
  const config = useMemo(() => ({
    theme: user.role === 'admin' ? 'dark' : 'light',
    permissions: ['read', 'write']
  }), [user.role]);

  useEffect(() => {
    // Efek ini hanya dijalankan jika &#x3C;code>config&#x3C;/code> berubah
    console.log('Config diperbarui:', config);
  }, [config]);

  return &#x3C;div>Profil Pengguna&#x3C;/div>;
}
</code></pre>
<hr>
<h3><strong>3. Kapan Menggunakan useMemo?</strong></h3>
<ul>
<li><strong>Komputasi intensif</strong> (misal: sorting/filter data besar).</li>
<li><strong>Mengirim nilai ke komponen anak yang di-optimasi</strong> (seperti React.memo).</li>
<li><strong>Mencegah re-render tidak perlu</strong> karena perubahan referensi objek/array.</li>
</ul>
<hr>
<h3><strong>4. Perbedaan useMemo vs useCallback</strong></h3>
<div class="overflow-x-auto bg-lime-100"><table>
<thead>
<tr>
<th><strong>Kriteria</strong></th>
<th>useMemo</th>
<th>useCallback</th>
</tr>
</thead>
<tbody>
<tr>
<td><strong>Return Value</strong></td>
<td>Nilai hasil fungsi</td>
<td>Fungsi itu sendiri</td>
</tr>
<tr>
<td><strong>Contoh</strong></td>
<td>useMemo(() => [1,2,3], [])</td>
<td>useCallback(() => {}, [])</td>
</tr>
<tr>
<td><strong>Penggunaan</strong></td>
<td>Untuk memoize nilai</td>
<td>Untuk memoize fungsi</td>
</tr>
</tbody>
</table></div>
<hr>
<h3><strong>5. Best Practices</strong></h3>
<ol>
<li>
<p><strong>Jangan Overuse</strong><br>
Gunakan hanya untuk komputasi yang benar-benar berat. Overuse bisa menambah kompleksitas.</p>
</li>
<li>
<p><strong>Perhatikan Dependency Array</strong><br>
Pastikan semua variabel yang digunakan di dalam fungsi dimasukkan ke dependency array:</p>
<pre><code class="language-javascript">// ‚ùå Salah (stale data jika &#x3C;code>price&#x3C;/code> berubah)
const total = useMemo(() => price * quantity, []);

// ‚úÖ Benar
const total = useMemo(() => price * quantity, [price, quantity]);
</code></pre>
</li>
<li>
<p><strong>Pisahkan Logika Komputasi</strong><br>
Untuk kode yang lebih bersih:</p>
<pre><code class="language-javascript">const calculateTotal = (price, quantity) => price * quantity;
const total = useMemo(() => calculateTotal(price, quantity), [price, quantity]);
</code></pre>
</li>
</ol>
<hr>
<h3><strong>6. Contoh Kasus: Kalkulasi Kompleks</strong></h3>
<pre><code class="language-jsx">function Fibonacci({ n }) {
  const fibNumber = useMemo(() => {
    const calculateFib = (num) => {
      if (num &#x3C;= 1) return num;
      return calculateFib(num - 1) + calculateFib(num - 2);
    };
    return calculateFib(n); // Komputasi berat untuk n besar
  }, [n]);

  return &#x3C;p>Fibonacci ke-{n}: {fibNumber}&#x3C;/p>;
}
</code></pre>
<hr>
<h3><strong>7. Kesalahan Umum</strong></h3>
<h4>a. <strong>Lupa Dependency Array</strong></h4>
<p>Nilai akan dihitung ulang setiap render:</p>
<pre><code class="language-javascript">const total = useMemo(() => price * quantity); // ‚ùå
</code></pre>
<h4>b. <strong>Menyimpan Nilai Sederhana</strong></h4>
<p>Tidak perlu memoize operasi trivial:</p>
<pre><code class="language-javascript">const total = useMemo(() => 10 + 20, []); // ‚ùå Tidak perlu
</code></pre>
<h4>c. <strong>Mengubah Nilai Langsung</strong></h4>
<p>Hindari mutasi objek/array:</p>
<pre><code class="language-javascript">const user = useMemo(() => {
  const newUser = { ...prevUser }; // ‚úÖ Salin objek
  newUser.age = 30;
  return newUser;
}, [prevUser]);
</code></pre>
<hr>
<h3><strong>Kesimpulan</strong></h3>
<p>Gunakan useMemo untuk:</p>
<ul>
<li><strong>Mengoptimalkan komputasi berat</strong>.</li>
<li><strong>Mencegah re-render komponen anak</strong> yang bergantung pada nilai kompleks.</li>
<li><strong>Mempertahankan referensi objek/array</strong> yang stabil.</li>
</ul>
<p>Hindari penggunaannya untuk nilai sederhana atau operasi trivial. Selalu evaluasi apakah optimasi ini benar-benar diperlukan! üöÄ</p>
