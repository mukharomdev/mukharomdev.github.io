e:Te68,<p><code>useRef</code> adalah hook di React yang digunakan untuk membuat referensi yang dapat dimutasi (mutable) dan bertahan sepanjang siklus hidup komponen. Berikut penjelasan lengkap beserta contoh penggunaannya:</p>
<h3>1. <strong>Mengakses DOM Element Langsung</strong></h3>
<p><code>useRef</code> sering digunakan untuk mengakses elemen DOM secara langsung, seperti fokus pada input.</p>
<pre><code class="language-jsx">import { useRef } from 'react';

function InputFocus() {
  // 1. Buat ref
  const inputRef = useRef(null);

  const handleClick = () => {
    // 3. Akses DOM element via ref.current
    inputRef.current.focus();
  };

  return (
    &#x3C;div>
      {/* 2. Attach ref ke elemen JSX */}
      &#x3C;input ref={inputRef} type="text" />
      &#x3C;button onClick={handleClick}>Fokus Input&#x3C;/button>
    &#x3C;/div>
  );
}
</code></pre>
<h3>2. <strong>Menyimpan Nilai yang Tidak Menyebabkan Re-render</strong></h3>
<p><code>useRef</code> bisa menyimpan nilai (seperti variabel) yang tetap bertahan antar-render, tetapi perubahan tidak memicu re-render.</p>
<pre><code class="language-jsx">import { useRef, useState, useEffect } from 'react';

function Timer() {
  const [count, setCount] = useState(0);
  const intervalRef = useRef(); // Simpan ID interval

  useEffect(() => {
    intervalRef.current = setInterval(() => {
      setCount((prev) => prev + 1);
    }, 1000);

    return () => clearInterval(intervalRef.current);
  }, []);

  const stopTimer = () => {
    clearInterval(intervalRef.current);
  };

  return (
    &#x3C;div>
      &#x3C;p>Hitungan: {count}&#x3C;/p>
      &#x3C;button onClick={stopTimer}>Hentikan Timer&#x3C;/button>
    &#x3C;/div>
  );
}
</code></pre>
<h3>3. <strong>Menyimpan Nilai Sebelumnya (Previous Value)</strong></h3>
<p>Contoh menyimpan nilai state sebelumnya:</p>
<pre><code class="language-jsx">import { useRef, useEffect, useState } from 'react';

function PreviousValue() {
  const [name, setName] = useState('');
  const prevNameRef = useRef('');

  useEffect(() => {
    prevNameRef.current = name; // Update ref setelah render
  }, [name]);

  return (
    &#x3C;div>
      &#x3C;input
        type="text"
        value={name}
        onChange={(e) => setName(e.target.value)}
      />
      &#x3C;p>Sekarang: {name}&#x3C;/p>
      &#x3C;p>Sebelumnya: {prevNameRef.current}&#x3C;/p>
    &#x3C;/div>
  );
}
</code></pre>
<h3>Perbedaan <code>useRef</code> vs. <code>useState</code></h3>
<div class="overflow-x-auto bg-lime-100"><table>
<thead>
<tr>
<th><strong>Fitur</strong></th>
<th><strong>useRef</strong></th>
<th><strong>useState</strong></th>
</tr>
</thead>
<tbody>
<tr>
<td>Re-render</td>
<td>Tidak memicu re-render</td>
<td>Memicu re-render</td>
</tr>
<tr>
<td>Nilai</td>
<td>Mutable (bisa diubah langsung)</td>
<td>Immutable (diubah via setState)</td>
</tr>
<tr>
<td>Penggunaan</td>
<td>Akses DOM, nilai persist</td>
<td>Data yang memengaruhi UI</td>
</tr>
</tbody>
</table></div>
<h3>Catatan Penting:</h3>
<ul>
<li><strong>Nilai <code>ref.current</code></strong> diinisialisasi saat komponen dipasang (mount) dan tetap ada hingga komponen di-unmount.</li>
<li><strong>Perubahan <code>ref.current</code> bersifat sinkronus</strong> (langsung terupdate).</li>
<li><strong>Jangan gunakan ref untuk nilai yang memengaruhi UI</strong> (gunakan <code>useState</code> atau <code>useEffect</code>).</li>
</ul>
<h3>Kapan Menggunakan <code>useRef</code>?</h3>
<ul>
<li>Manipulasi DOM langsung (fokus, scroll, animasi).</li>
<li>Menyimpan nilai seperti timer ID, instance library eksternal, atau variabel yang perlu persist.</li>
<li>Menyimpan nilai sebelumnya (previous state) untuk keperluan komparasi.</li>
</ul>
